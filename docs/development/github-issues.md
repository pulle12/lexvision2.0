# GitHub Issues

Issues werden für Projektmanagement und Aufgabenverfolgung verwendet.

Jede Aufgabe soll als eigenes Issue erfasst werden.

Beispiele für Issue-Typen (Labels):

- feature
- bug
- documentation
- research

Issues werden mit GitHub Projects organisiert.

Zweck:

- Entwicklungsfortschritt nachverfolgen
- Sprint-Backlog verwalten
- Aufgaben und Diskussionen dokumentieren
