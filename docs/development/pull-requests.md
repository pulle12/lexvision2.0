# Pull-Request-Richtlinien

Alle Änderungen sollen über Pull Requests gemergt werden.

## Ablauf

1 Feature-Branch erstellen  
2 Branch zu GitHub pushen  
3 Pull Request eröffnen  
4 Review anfordern  
5 Nach Freigabe mergen

## PR-Beschreibung

Jeder PR sollte enthalten:

- kurze Beschreibung
- verlinktes Issue
- Zusammenfassung der Änderungen

Beispiel:

Closes #12  
Adds evidence upload endpoint.
