# Branch-Schutzregeln

Der Branch `main` ist über GitHub Branch Protection Rules geschützt.

Die konkrete Konfiguration wird durch die Projektleitung im Repository vorgenommen.

Branch-Muster:

main

Einstellungen (Zielbild):

- Pull Request vor dem Mergen erforderlich
- Mindestens 1 Approval notwendig
- Status-Checks müssen erfolgreich sein
- Direkte Pushes auf main sind eingeschränkt

Nur die Projektleitung kann in begründeten Ausnahmefällen von diesen Regeln abweichen.

Zweck:

- unbeabsichtigte Commits auf main verhindern
- Code-Review sicherstellen
- einen stabilen main-Branch erhalten
